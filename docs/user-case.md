# UC-1: Регистрация и вход

## TC-1.1 Успешная регистрация
**Предусловие:** пользователь не зарегистрирован  

**Шаги:**
1. Ввести валидный email  
2. Ввести пароль (валидный по требованиям)  
3. Нажать "Регистрация"  

**Ожидаемый результат:**
- Аккаунт создан  
- Возвращается JWT токен  
- Пользователь авторизован  


## TC-1.2 Регистрация с уже существующим email
**Шаги:**
1. Ввести email, который уже есть в системе  
2. Ввести пароль  
3. Нажать "Регистрация"  

**Ожидаемый результат:**
- Ошибка "Email уже используется"  
- Аккаунт не создается  


## TC-1.3 Успешный вход
**Предусловие:** пользователь зарегистрирован  

**Шаги:**
1. Ввести корректный email  
2. Ввести корректный пароль  
3. Нажать "Войти"  

**Ожидаемый результат:**
- Возвращается JWT токен  
- Пользователь авторизован  


## TC-1.4 Неверный пароль
**Шаги:**
1. Ввести корректный email  
2. Ввести неверный пароль  

**Ожидаемый результат:**
- Ошибка авторизации  
- Токен не выдается

# UC-2: Создание сервера

## TC-2.1 Успешное создание сервера

**Шаги:**

3. Подтвердить  

**Ожидаемый результат:**

**Шаги:**
1. Открыть сервер  
2. Создать текстовый канал  

**Ожидаемый результат:**
- Канал появляется в списке  


## TC-2.3 Генерация инвайт-ссылки

2. Нажать "Создать инвайт"  

**Ожидаемый результат:**
- Сгенерирована уникальная ссылка  
- Ссылка работает  


## TC-2.4 Переход по инвайт-ссылке
**Шаги:**
1. Перейти по ссылке  

**Ожидаемый результат:**
- Пользователь добавлен в сервер

# UC-3: Отправка сообщения

## TC-3.1 Отправка сообщения
**Предусловие:** пользователь в канале  

**Шаги:**
1. Ввести текст  
2. Нажать "Отправить"  

**Ожидаемый результат:**
- Сообщение сохранено на сервере  
- Сообщение отображается у отправителя  
- Сообщение приходит другим пользователям через WebSocket  


## TC-3.2 Получение сообщения другим пользователем
**Предусловие:** второй пользователь онлайн  

**Ожидаемый результат:**
- Сообщение появляется без перезагрузки  


## TC-3.3 Отправка пустого сообщения
**Шаги:**
1. Отправить пустую строку  

**Ожидаемый результат:**
- Ошибка / сообщение не отправляется  


## TC-3.4 Потеря соединения WebSocket
**Шаги:**
1. Отключить интернет  
2. Отправить сообщение  

**Ожидаемый результат:**
- Сообщение не отправляется  
- Есть уведомление об ошибке

# UC-6: Онлайн-статусы

## TC-6.1 Пользователь онлайн
**Шаги:**
1. Пользователь отправляет heartbeat  

**Ожидаемый результат:**
- Статус = онлайн  


## TC-6.2 Пользователь становится офлайн
**Шаги:**
1. Прекратить отправку heartbeat  

**Ожидаемый результат:**
- Через 30 секунд статус = офлайн  


## TC-6.3 Регулярный heartbeat
**Шаги:**
1. Отправлять heartbeat каждые 20 секунд  

**Ожидаемый результат:**
- Пользователь остается онлайн  


## TC-6.4 Видимость статуса
**Предусловие:** два пользователя без общего сервера  

**Ожидаемый результат:**
- Статус не отображается
